# フォトウォークにおけるフォトスポットと撮影者の同時クラスタリング / Simultaneous Clustering of Photo Spots and Photographers in Photowalks

## Recap

- フォトウォークの定義
  - カメラを持って街を歩き，興味を引いたものを撮影する行為
- 個人的穴場フォトスポットの定義
  - 発見的: 一般的な人気が高いが，自分と同じ嗜好を持った人からの人気は低い
  - 独占的: 一般的な人気は低いが，自分と同じ嗜好を持った人からの人気は高い
- フォトウォークにおいて個人的穴場を提案する意義
  - 自分の感性に合った被写体との出会いを促進
  - フォトウォーク本来の「発見する楽しさ」を増幅
- 問題
  - フォトウォークにおいて個人的穴場を発見する方法がない

## Background

- 着想
  - 観光スポットと訪問者の同時クラスタリング手法をフォトウォークに応用
    - フォトスポットと撮影者の同時クラスタリングにより個人的穴場を抽出
- 必要なデータ
  - フォトウォークデータ: 位置情報，被写体情報付きの撮影データ
- 問題
  - フォトウォークデータを入手することが難しい
    - 移動経路を含む密な撮影データが必要（写真共有SNSの疎なデータでは不十分）
    - 個人の移動履歴を含むため，プライバシーの観点からデータ公開が困難
    - フォトウォーク専用のプラットフォームが存在せず，Web上からの収集も困難
  - データそのものを生成することで，これら全ての問題を回避

## Purpose

- 研究全体の流れと本研究の位置づけ
  - 問題（個人的穴場の発見方法がない）→ 着想（同時クラスタリングの応用）→ **本研究: データ生成**（LLMによるフォトウォークデータ生成）→ 今後（推薦システムの構築と評価）
- 目的
  - 個人的穴場フォトスポット抽出手法の検証に向けたデータセットの構築
- 提案手法
  - フォトウォークデータに求める要件を言語化し，Gemini API の構造化出力機能を用いてデータ生成

## Related Works

- LLMによる合成データ生成の信頼性
  - アンケート回答の再現性
    - Argyle et al. (2023) "Out of One, Many": 多様な背景設定を与えることで実際の人間によるアンケート回答の分布を高い精度で再現できる
  - 行動シミュレーションの信頼性
    - Park et al. (2023) "Generative Agents": 人間らしい社会的振る舞いや長期的な計画行動を自律的に生成できる
- 地理空間情報の生成能力
  - Hasan et al. (2025) "MapAgent": 複雑な地理的クエリをサブゴールに分解し階層的に処理，専用エージェントが複数のAPIを適応的に並列実行
  - Yu et al. (2025) "Spatial-RAG": 空間データベースによる検索と意味的類似度による検索を統合し，空間的な質問応答の精度を大幅に改善

## Method

- フォトウォークデータ生成の要件と手法選定
  - フォトウォークの地理的・時間的文脈を理解した上でデータを生成する必要がある
  - スポットの座標・被写体カテゴリ・ユーザ嗜好を含む構造化データが必要
  - 推薦手法の検証に十分な規模のデータを生成する必要がある
  - Google Gemini API の構造化出力機能を採用
    - 地理情報を含む長いプロンプトを処理できる（コンテキスト容量）
    - JSON スキーマに厳密に従った出力が可能（構造化出力）
- スキーマ設計
  - Session（1回のフォトウォーク）: ユーザID, Recordリスト
  - Record（各撮影地点）:
    - 緯度・経度 → スポットのクラスタリングに使用
    - 被写体カテゴリ → ユーザ嗜好の抽出に使用
    - 時刻 → 行動パターンの分析に使用
- プロンプト設計
  - エージェントの役割を明確化
  - 周回性: 実際のフォトウォークの行動特性を再現
  - 地理的整合性: 実在する道路上の座標を生成
  - 生成条件（エリア名，データ規模）
  ```text
  あなたは<エリア名>エリアをフォトウォークするエージェントである．
  以下の条件に従って，フォトウォークを行いなさい．
  - <画像の枚数>枚程度の画像を撮影すること．
  - 徒歩<1セッションの時間>分程度の距離を歩くこと．
  - 出発地点と到着地点が同一であること．
  - 撮影する場所が地図の道路上であること．
  ```

## Implementation

- Python と Google GenAI SDK を使用（並行処理で大量生成）
- モデル選定：フォトウォークデータの制約を遵守できるかで比較
  - Gemini 2.5 Flash Lite: 周回性を遵守できない → 不採用
  - Gemini 2.5 Flash: 地理的整合性を遵守できない → 不採用
  - Gemini 3 Flash Preview: 両制約を遵守 → 採用

## Experiment

- 生成条件
  - 対象エリア
    - 北海道函館市（美原・神山・富岡町・中道エリア）
  - 行動制約
    - 徒歩30分程度の散歩コース
    - 10〜30枚程度の画像撮影
  - データ規模
    - 10人の撮影者 × 5セッション = 50セッション
- 生成結果
  - 生成したデータを地図上にプロット

## Evaluation

- スキーマ準拠性
  - スキーマに準拠した出力を得られた
- 地理的整合性
  - 道路上の存在
    - 厳密には道路上ではない撮影地点がある
    - 小数点以下第3位までの精度であり100m程度の誤差が生じる
    - フィールドの説明に加えたが遵守されない
    - 建物・公園内など道路外が妥当な地点もある
  - エリア内包
    - 生成した全てのデータが概ね指定したエリア内に存在する
- 行動制約の遵守
  - 周回性
    - スタート地点とゴール地点が同一地点になるようなセッションが生成される
  - 距離・時間
    - 1セッションの地点間距離の合計は 1.55 - 5.04 km
    - 4.8 km/h で歩いた場合 19 - 63 分
    - 徒歩30分程度という条件を満たしていない
    - 道路に沿った距離を計測した場合はさらに長くなる
  - 撮影枚数
    - 1セッションで撮影された画像は 12 - 21 枚
    - 10 - 30 枚という条件を概ね満たしている
- 多様性
  - 経路の多様性
    - 指定したエリア全域にわたって多様な撮影地点が生成される
  - 被写体の多様性
    - 偏りがなく多様なカテゴリの撮影データが生成される
    - 撮影者による偏りがない（嗜好分析には偏りが必要 → 課題）
- 問題と解決策
  - 座標が小数点以下第3位までの精度
    - プロンプトでの改善は困難
    - 道路外でも妥当な地点の扱いを整理した上で，座標補正の手法を検討
  - 道路に沿った距離が指定した時間を超える
    - 撮影地点以外の移動経路も生成させるようにする
  - 撮影者による被写体カテゴリの偏りがない
    - プロンプトでエージェントに多様なペルソナを与えて興味を持たせるようにする

## Plan

- 本研究の位置づけ
  - データ生成は推薦システム構築に向けた第一段階
- 短期: データ生成の改善
  - 座標精度の向上（座標補正の導入を検討）
  - 個人の嗜好の反映（ペルソナ付与による撮影傾向の差異化）
  - 移動経路の生成（撮影地点間の経由点も出力）
- 中長期: 推薦システムの構築と評価
  - 生成データを用いた同時クラスタリングによる個人的穴場の抽出
  - 被験者実験による推薦精度の検証

## Summary

- 背景
  - フォトウォークにおいて個人の感性に合った「個人的穴場」の推薦が必要
  - 推薦システムの構築に必要なフォトウォークデータ（位置・被写体情報）の入手が困難
- 目的
  - 個人的穴場抽出に向けた，LLMによるフォトウォークデータの生成
- 提案手法
  - Gemini APIの構造化出力を活用し，地理的整合性と行動制約を満たすデータを生成
- 結論
  - 地理的整合性と行動制約を概ね満たすフォトウォークデータを生成できた
  - 位置情報精度の向上、移動経路の生成、嗜好の反映などの課題が残った
- 今後の方針
  - 個人の嗜好を反映したデータ生成と，それを用いた推薦システムの構築・評価
