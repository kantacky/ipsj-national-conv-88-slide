# 本研究のデータ生成に求められる要件

- フォトウォークの**地理的・時間的文脈**を理解した上でデータを生成する必要がある
- スポットの**座標・被写体カテゴリ**を含む構造化データが必要
- 推薦手法の検証に**十分な規模のデータ**を生成する必要がある

# Google Gemini API の構造化出力機能を採用

- JSON スキーマに厳密に従った出力が可能（構造化出力）

# スキーマ設計：必要な情報を構造化

- Session：1回のフォトウォーク
  - ユーザID
  - Recordリスト
- Record：各撮影データ
  - 時刻：データの順序を決定
  - 緯度・経度：撮影地点
  - 被写体カテゴリ：嗜好を抽出
    - landscape, building, vehicle, person, animal, plant

# プロンプトに含める条件

- エージェントの役割を明確化
- **周回性**: 実際のフォトウォークの行動特性を再現
- **地理的整合性**: 実在する道路上の座標を生成
- 生成条件（エリア名，データ規模）

</div>
<div>

# プロンプト

```text
あなたは<エリア名>エリアをフォトウォークするエージェントである．
以下の条件に従って，フォトウォークを行いなさい．
- <画像の枚数>枚程度の画像を撮影すること．
- 徒歩<1セッションの時間>分程度の距離を歩くこと．
- 出発地点と到着地点が同一であること．
- 撮影する場所が地図の道路上であること．
```
