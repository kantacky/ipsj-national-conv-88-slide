# 本研究のデータ生成に求められる要件

- フォトウォークの**地理的・時間的文脈**を理解した上でデータを生成する必要がある
- スポットの**座標・被写体カテゴリ・ユーザ嗜好**を含む構造化データが必要
- 推薦手法の検証に十分な**規模のデータ**を生成する必要がある

# → Google Gemini API の構造化出力機能を採用

- 地理情報を含む長いプロンプトを処理できる（コンテキスト容量）
- JSON スキーマに厳密に従った出力が可能（構造化出力）

# スキーマ設計：推薦手法に必要な情報を構造化

- **Session**（1回のフォトウォーク）: ユーザID, Recordリスト
- **Record**（各撮影地点）:
  - 緯度・経度 → **スポットのクラスタリング**に使用
  - 被写体カテゴリ → **ユーザ嗜好の抽出**に使用
  - 時刻 → 行動パターンの分析に使用

# プロンプトに含める条件

- エージェントの役割を明確化
- **周回性**: 実際のフォトウォークの行動特性を再現
- **地理的整合性**: 実在する道路上の座標を生成
- 生成条件（エリア名，データ規模）

</div>
<div>

# プロンプト

```text
あなたは<エリア名>エリアをフォトウォークするエージェントである．
以下の条件に従って，フォトウォークを行いなさい．
- <画像の枚数>枚程度の画像を撮影すること．
- 徒歩<1セッションの時間>分程度の距離を歩くこと．
- 出発地点と到着地点が同一であること．
- 撮影する場所が地図の道路上であること．
```
